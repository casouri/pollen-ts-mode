# Environment Setup

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| Node.js | ≥ 18 | Run tree-sitter CLI and build scripts |
| npm | ≥ 9 | Package management |
| tree-sitter CLI | latest | Generate parser, run tests, parse files |
| C compiler (gcc/clang) | any recent | Compile generated C parser |
| git | any | Version control |

### Install tree-sitter CLI

```bash
npm install -g tree-sitter-cli
# verify:
tree-sitter --version
```

### Verify C compiler

```bash
cc --version
```

---

## Project Initialization

### 1. Create project directory

```bash
mkdir tree-sitter-pollen
cd tree-sitter-pollen
git init
```

### 2. `package.json`

```json
{
  "name": "tree-sitter-pollen",
  "version": "0.1.0",
  "description": "Tree-sitter grammar for the Pollen publishing language",
  "main": "bindings/node",
  "scripts": {
    "build": "tree-sitter generate",
    "test": "tree-sitter test",
    "parse": "tree-sitter parse"
  },
  "dependencies": {
    "nan": "^2.18.0"
  },
  "devDependencies": {
    "tree-sitter-cli": "latest"
  },
  "tree-sitter": [
    {
      "scope": "source.pollen",
      "file-types": ["pm", "pp", "p"],
      "injection-regex": "pollen"
    }
  ]
}
```

### 3. `binding.gyp`

```json
{
  "targets": [
    {
      "target_name": "tree_sitter_pollen_binding",
      "include_dirs": [
        "<!(node -e \"require('nan')\")",
        "node_modules/tree-sitter/src"
      ],
      "sources": [
        "bindings/node/binding.cc",
        "src/parser.c"
      ],
      "cflags_cc": ["-std=c++14"]
    }
  ]
}
```

### 4. `.gitignore`

```
node_modules/
src/
build/
```

> `src/` is generated by `tree-sitter generate` — do not edit by hand.

### 5. Initial `grammar.js` (skeleton)

```js
module.exports = grammar({
  name: 'pollen',

  rules: {
    source_file: $ => seq(),
  }
});
```

### 6. Generate and verify

```bash
npm install
tree-sitter generate    # produces src/parser.c
tree-sitter test        # runs test/corpus/ tests (empty at this point)
```

## Useful Commands

```bash
# Regenerate parser after editing grammar.js
tree-sitter generate

# Run all corpus tests
tree-sitter test

# Run tests matching a pattern
tree-sitter test -i "Tag with body"

# Auto-update expected output in corpus files
tree-sitter test -u

# Parse a file and print the syntax tree
tree-sitter parse examples/ts.html.pm

# Parse and show byte ranges
tree-sitter parse --stat examples/ts.html.pm

# Interactive REPL-style debugging
tree-sitter query '(tag_expression) @tag' examples/ts.html.pm
```

## Editor Integration (optional but helpful)

Install the `nvim-treesitter` or VS Code tree-sitter extension, point it at this local grammar for syntax highlighting while developing.

For Emacs:
```elisp
(add-to-list 'treesit-language-source-alist
             '(pollen "https://github.com/yourname/tree-sitter-pollen"))
```
